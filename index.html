Objetivos

Es un sistema que elimina información redundante o no deseada de flujos de información mediantes medios semi-automatizados o automatizados. Esto con el fin de de evitar que el consultor sea sobrecargado por el exceso de información y disminuir el ruido en la consulta.

“Facilitar la búsqueda de información útil.”

“Organizar información de forma comprensible y correcta.”

“Disminuir la información no relevante entregada al usuario.”

“Tratar que la información dada al usuario sea de utilidad.”

“Mejorar la experiencia para el usuario.”

“Sobresalir entre otras empresas al brindar un mejor servicio.”

Máquinas de Vectores de Soporte

Son algoritmos de aprendizaje automático supervisado, donde trazamos cada elemento de datos como un punto en el espacio n-dimensional (donde n es el número de entidades que tiene) con el valor de cada entidad como el valor de una coordenada en particular. Luego, realizamos la clasificación encontrando el hiperplano que diferencia a las dos clases.

Esto permite que al incluir un nuevo elemento este pueda ser clasificado como de un elemento relevante o no relevante, al determinar que características tienen cada uno de estos en común.

Preprocesar el documento a ser juzgado para obtener la lista de los términos utilizados, utilizando el mismo método del utilizado para alimentar el filtro.

Para cada término del documento, sumar su peso y el factor normalizado al contenido de dos acumuladores inicializados en 0.

Se divide el valor en el acumulador de pesos entre el valor del acumulador de factores normalizados.

Si el resultado es mayor del valor de la variable “threshold”, se retorna el valor como la calificación del filtro para el documento. En otro caso retorna 0 para saber que el documento no puede ser juzgado.

Pros

Funciona muy bien con un claro margen de separación.

Es eficaz en espacios de alta dimensión.

Es efectivo en casos donde el número de dimensiones es mayor que el número de muestras.

Utiliza un subconjunto de puntos de entrenamiento en la función de decisión (llamados vectores de soporte), por lo que también es eficiente en memoria.

Contras

No tiene un buen rendimiento cuando tenemos un gran conjunto de datos porque el tiempo de entrenamiento requerido es mayor.

Tampoco tiene un buen rendimiento, cuando el conjunto de datos tiene más ruido, es decir, las clases de destino se superponen.

SVM no proporciona directamente estimaciones de probabilidad, estas se calculan utilizando una costosa validación cruzada de cinco veces.

Filtrado Bayesiano

Usado comúnmente en filtros antispam, este filtrado que usa palabras particulares que tienen determinadas probabilidades de ocurrir en correos spam o sitios de dudosa fiabilidad y que puedan estar en correos o sitios legítimos.


Pr(S|W)


Es la probabilidad de que sea un correo no deseado o un sitio poco confiable.

Pr(S)


Es la probabilidad general de que cualquier mensaje dado es spam o un sitio poco confiable.

Pr(W|S)


Es la probabilidad de que una determinada palabra aparezca en los spam o sitios poco confiables.


Pr(H)


Es la probabilidad general de que cualquier mensaje dado no es spam (es "legítimo") o un sitio confiable.

Pr(W|H)


Es la probabilidad de que una determinada palabra no aparezca en los spam o sitios poco confiables.

Filtrado colaborativo

Usado en sistemas de recomendación, permite filtrar sobre los posibles gustos o intereses de un usuario mediante la recopilación de las preferencias o información obtenida de los intereses de muchos otros usuarios. Esto basándose en la suposición que: ”Si la persona A tiene iguales preferencias sobre un determinado tema que la persona B, es muy probable que tengan la misma opinión sobre otro tema elegido al azar”, esto mediante el discernimiento de patrones obtenidos de múltiples fuentes o con un gran conjunto de datos sobre el grupo objetivo.

Requerimientos

Dado que el sistema se basa en la opinión de los usuarios, es necesaria el que estos estén continuamente proporcionando retroalimentación al sistema y que esta participación pueda ser diferenciada de la de otros usuarios.

Participación activa de usuarios

Es necesario que el usuario brinde su opinión sobre el grado de interés o gusto hacia los resultados entregados al usuario.

Forma de representar los intereses de los usuarios

Es necesario el disponer de un algoritmo que permita asociar a personas con intereses o gustos similares para permitir realizar predicciones de gustos.

Algoritmo de emparejamiento

Ejemplo

Se tiene un grupo de individuos con los siguientes gustos
Se crea una matriz que lista los gustos de los individuos
Se determina quienes tienen más similitudes con el usuario de interés
Se procede a calcular el gusto más posible que el usuario de interés pueda tener, basándose en gustos e intereses de personas similares
Se entrega información al usuario dependiendo sus intereses predichos

Problemas o desafíos

Estos sistemas necesitan de gran cantidad de datos para funcionar, siendo uno de los principales problemas en sistemas nuevos, ya que es necesario que suficientes usuarios brinden sus opiniones para crear un matriz de preferencias precisa. También sucede esto cuando se agregan nuevos elementos a calificar.

Falta de datos

Son usuarios cuyas opiniones no están de acuerdo o en desacuerdo con las de ningún grupo de usuarios.

Usuarios ”ovejas grises”

Dado un gran número de usuarios y de elementos calificables pueden llegar a ser demasiado complejo para la mayoría de algoritmos
(pueden ser millones de usuarios y de elementos).

Escalabilidad

Las artículos nuevos pueden llegar a no ser calificados por la cantidad suficiente de individuos para ser empezado a ser recomendado.

Diversidad

Los dueños de un elemento podría usar usuarios para aumentar su propia popularidad y disminuir la de la competencia.

Redes Neuronales

Dado que las redes neuronales pueden predicciones a partir de datos previos disponibles, estas pueden entregarse para la extracción de datos útiles. Al usar redes neuronales para filtrado de información es recomendado usar un sistema basado en el entrenamiento supervisado.

El proceso por el que se realiza el filtrado consiste en dos fases:

  • El entrenamiento: es la fase en la que red es entrenada para distinguir artículos relevantes de los no relevantes para la cual puede ser necesaria una gran cantidad de data y usualmente es el proceso de mayor duración, en esta se dan las siguientes operaciones:
    • Calificación del artículo: cada artículo debe de ser calificado en consideración al valor que este tenga para el usuario.
    • Se filtra las “stop words” de cada artículo.
    • Se asigna un peso a cada término de cada artículo.
    • Se envía grupos de términos a la red para entregarla, modificando entre cada envío el valor de los pesos de las neuronas que componen la red para así ajustar la salida al valor real.
  • Utilización: una vez entrenada la red esta puede reconocer los artículos que pueden ser considerados de interés para los mismos usuarios cuyas opiniones fueron utilizadas para entrenar la red
